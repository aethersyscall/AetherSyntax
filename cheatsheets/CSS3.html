CSS3 Logic

The Visual Language of the Web.

CSS (Cascading Style Sheets) controls the presentation of your document. While HTML defines the meaning, CSS defines the feeling. It handles layout, color, typography, and motion.

I. Novice (The Basics)

Understanding how to target elements and how space works is the prerequisite for everything else.

The Syntax

A CSS rule consists of a selector and a declaration block.

css
/* Selector { Property: Value; } */
h1 {
  color: #9580FF;
  font-size: 2rem;
}

Selectors

How we tell the browser which element to style.

  • Type Selector: Targets all elements of a type (e.g., p, div). Low specificity.
  • Class Selector (.): Targets elements with a specific class attribute. Preferred method.
  • ID Selector (#): Targets a unique element. High specificity (avoid for general styling).
css
/* Target all <button> tags */
button {
  background: none;
}

/* Target <div class="card"> */
.card {
  border: 1px solid #2E283B;
}

/* Target <div id="app"> */
#app {
  min-height: 100vh;
}

The Box Model

Every element on a webpage is a rectangular box. Understanding this is non-negotiable.

  1. Content: The actual text or image.
  2. Padding: Space inside the border (clears space around content).
  3. Border: The line wrapping the padding.
  4. Margin: Space outside the border (pushes other elements away).

Pro Tip: Always apply this reset to prevent math headaches. It makes the width include padding and border.

css
* {
  box-sizing: border-box;
}

II. Apprentice (Flexbox & Typography)

Moving elements around and making text readable.

Flexbox (1-Dimensional Layout)

Flexbox is designed for laying out items in a single row or column.

css
.nav-bar {
  display: flex;
  justify-content: space-between; /* Spacing along main axis */
  align-items: center;          /* Alignment along cross axis */
  gap: 1rem;                    /* Space between items */
}

Typography

Good design is 90% typography.

css
body {
  font-family: 'Inter', sans-serif;
  /* Unitless line-height is relative to font-size (1.5 = 150%) */
  line-height: 1.6; 
  /* Use 'rem' for accessibility (relative to root font size) */
  font-size: 1rem;
  color: #EDE8F5;
}

III. Journeyman (Grid & Positioning)

Building complex structures and handling responsiveness.

CSS Grid (2-Dimensional Layout)

Grid handles rows and columns simultaneously. It is the most powerful layout system available.

css
.dashboard-grid {
  display: grid;
  /* Create columns that fill space automatically, min 300px wide */
  grid-template-columns: repeat(auto-fill, minmax(300px, 1fr));
  gap: 2rem;
}

Positioning

Taking elements out of the normal document flow.

  • static: Default behavior.
  • relative: Position relative to itself. Unlocks z-index.
  • absolute: Position relative to the nearest positioned ancestor.
  • fixed: Stuck to the viewport (e.g., a modal overlay).
  • sticky: Toggles between relative and fixed based on scroll.
css
.header {
  position: sticky;
  top: 0;
  z-index: 100; /* Stacks above other content */
}

Media Queries (Responsiveness)

Changing styles based on device characteristics (Mobile-First approach).

css
/* Base styles are for Mobile */
.sidebar {
  display: none;
}

/* Overrides for Desktop */
@media (min-width: 768px) {
  .sidebar {
    display: block;
    width: 250px;
  }
}

IV. Expert (Variables & Polish)

Writing maintainable, scalable, and interactive styles.

CSS Variables (Custom Properties)

Define values once, use everywhere. This is the key to theming (like Eclipse/Bliss).

css
:root {
  --primary: #9580FF;
  --bg-color: #13111B;
  --spacing-md: 1.5rem;
}

.button {
  background-color: var(--primary);
  padding: var(--spacing-md);
}

Pseudo-Classes & Elements

Styling states and injecting content without HTML.

css
/* State changes */
a:hover { color: var(--primary); }
input:focus { outline: 2px solid var(--primary); }

/* Elements */
.card::before {
  content: ''; /* Required */
  position: absolute;
  top: 0;
  left: 0;
  width: 100%;
  height: 2px;
  background: linear-gradient(90deg, #FF70B8, #80FFEA);
}

Transitions & Animation

Smoothly animating changes.

css
.card {
  transition: transform 0.3s cubic-bezier(0.23, 1, 0.32, 1);
}

.card:hover {
  transform: translateY(-5px);
}

V. Grandmaster (Architecture & Math)

Writing CSS that survives large-scale applications.

The Cascade & Specificity

When two rules conflict, the browser calculates a score to decide which wins:

  1. Inline styles: style="..." (1000 points)
  2. ID selectors: #header (100 points)
  3. Classes/Attributes: .btn, [type="text"] (10 points)
  4. Elements: div, h1 (1 point)

Rule: Keep specificity low. Use classes. Avoid !important.

Modern Functions

Using math within CSS for fluid layouts.

css
h1 {
  /* Scale font smoothly between 2rem and 4rem based on viewport width */
  font-size: clamp(2rem, 5vw, 4rem);
}

.sidebar {
  /* Calculate width dynamically */
  width: calc(100% - 40px);
}

Architecture (BEM)

Block Element Modifier. A naming convention to prevent style leaks.

css
/* Block */
.card {} 

/* Element (dependent on block) */
.card__title {} 
.card__button {}

/* Modifier (variation) */
.card--featured {} 
.card__button--primary {}
