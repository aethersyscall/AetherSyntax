HTML5: The Structure

The Semantic Skeleton of the Web.

HTML (HyperText Markup Language) is not a programming language; it is a structural markup language. It defines the "what" of a webpage: this is a heading, this is an image, this is a navigation bar.

I. Novice (The Foundation)

Before you can paint with CSS or animate with JavaScript, you must build the skeleton.

The Boilerplate

Every valid HTML document requires a specific structure. The browser reads this top-to-bottom.

html
<!DOCTYPE html>
<html lang="en">
<head>
    <!-- Meta-data: The browser reads this, users don't see it -->
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>Page Title</title>
</head>
<body>
    <!-- Visible Content: Everything the user sees goes here -->
    <h1>Hello, World!</h1>
</body>
</html>

Tags & Attributes

HTML is made of elements. An element consists of a start tag, content, and an end tag.

  • Tags: <p> opens a paragraph. </p> closes it.
  • Attributes: Provide extra info inside the opening tag.
  • Void Elements: Tags that don't hold content and don't need closing (e.g., <img />, <br />, <input />).
html
<!-- Class: Reusable identifier for CSS styling -->
<p class="text-large">This is a paragraph.</p>

<!-- ID: Unique identifier for JS hooks or anchors -->
<div id="hero-section"></div>

<!-- Void Tag (Self-closing) -->
<hr />

II. Apprentice (Content & Media)

Once the shell exists, we populate it with information.

Typography

Hierarchy matters. Search engines use headings to understand your content structure.

html
<h1>Main Title (Only use one per page)</h1>
<h2>Section Heading</h2>
<h3>Subsection</h3>

<p>Standard paragraph text.</p>

<!-- Semantic Emphasis -->
<strong>Bold (Important)</strong>
<em>Italic (Emphasized)</em>

Hyperlinks (Anchors)

The <a> tag connects the web.

html
<!-- External Link (New Tab) -->
<a href="https://google.com" target="_blank" rel="noopener">Google</a>

<!-- Internal Link -->
<a href="/about.html">About Us</a>

<!-- Jump Link (Scrolls to id="contact") -->
<a href="#contact">Contact</a>

Images

Images require the src (source) and alt (alternative text) attributes. The alt text is mandatory for accessibility (screen readers) and SEO.

html
<img 
  src="logo.png" 
  alt="AetherSyntax Logo" 
  width="200" 
  height="50" 
  loading="lazy" 
/>

Lists

Grouping related items.

html
<!-- Unordered List (Bullets) -->
<ul>
  <li>Rust</li>
  <li>Go</li>
  <li>C++</li>
</ul>

<!-- Ordered List (Numbers) -->
<ol>
  <li>Step One</li>
  <li>Step Two</li>
</ol>

III. Journeyman (Semantics & Layout)

A rookie uses <div> for everything. A master uses Semantic HTML. Semantic tags describe their meaning, not just their look. This is critical for Screen Readers and SEO bots.

The Semantic Layout

Instead of <div id="header">, use <header>.

html
<header>
  <nav>
    <!-- Navigation links -->
  </nav>
</header>

<main>
  <article>
    <!-- A self-contained piece of content (e.g., blog post) -->
    <h2>Blog Post Title</h2>
    <p>Content...</p>
  </article>

  <aside>
    <!-- Content related to the main content (Sidebar) -->
    <p>Related links...</p>
  </aside>
</main>

<footer>
  <!-- Copyright, Sitemap -->
</footer>

Block vs. Inline

  • Block: Starts on a new line and takes up full width (e.g., <div>, p, h1, section).
  • Inline: Sits within a line of text (e.g., span, a, strong, img).

IV. Expert (Forms & Interaction)

The primary way users send data to your server. Forms are complex state machines built into the browser.

The Form Wrapper

The <form> tag defines where data goes (action) and how it gets there (method).

html
<form action="/api/login" method="POST">
  <!-- Inputs go here -->
</form>

Inputs & Labels

Always connect an <input> to a <label> using for and id. This expands the click area and tells screen readers what the input is.

html
<div class="input-group">
  <label for="user-email">Email Address</label>
  <input 
    type="email" 
    id="user-email" 
    name="email" 
    placeholder="you@example.com" 
    required 
  />
</div>

<div class="input-group">
  <label for="user-pass">Password</label>
  <input type="password" id="user-pass" name="password" minlength="8" />
</div>

<!-- Buttons default to type="submit" inside forms -->
<button type="submit">Login</button>

Input Types

Using the correct type triggers the correct keyboard on mobile devices.

  • type="email": Shows @ symbol.
  • type="number": Shows number pad.
  • type="tel": Shows phone pad.
  • type="date": Shows native date picker.

V. Grandmaster (Meta & Accessibility)

The invisible layer that controls how the world perceives your site.

The Meta Layer (<head>)

Control how your link looks when shared on Twitter/Discord (Open Graph).

html
<!-- SEO Description -->
<meta name="description" content="The ultimate syntax guide for developers.">

<!-- Open Graph (Social Cards) -->
<meta property="og:title" content="AetherSyntax">
<meta property="og:image" content="https://aether.com/banner.png">
<meta property="og:type" content="website">

<!-- Viewport (Critical for Mobile Responsiveness) -->
<meta name="viewport" content="width=device-width, initial-scale=1.0">

Accessibility (A11y) & ARIA

HTML5 is accessible by default if used correctly. Sometimes you need to give the browser hints using ARIA (Accessible Rich Internet Applications).

html
<!-- A button that is icon-only needs a label for screen readers -->
<button aria-label="Close Menu">
  <svg>...</svg>
</button>

<!-- Hiding decorative elements from screen readers -->
<div class="decorative-circle" aria-hidden="true"></div>

<!-- Live Regions (Announce updates without page reload) -->
<div aria-live="polite">
  <span>User saved successfully.</span>
</div>

The DOM (Document Object Model)

HTML is just text until the browser parses it into the DOM. The DOM is a tree structure of objects.

  • HTML is the blueprint.
  • DOM is the house built from that blueprint.
  • JavaScript modifies the DOM (rearranging furniture), not the HTML file itself.
